_engine.css

A property-driven CSS design system. One formula for color, one formula for type. Everything inherits, composes, and adapts to dark mode automatically. 802 lines, ~27KB, zero dependencies, no build step.

The system is built on three CSS features: @property registrations for typed custom properties, OKLCH color math for perceptually uniform color derivation, and CSS pow() for fluid type scaling.

Philosophy

The five public knobs — --hue, --chroma, --surface, --contrast, --type — control the entire visual output. You never write a color value or a font-size. You set knobs and the engine derives everything.

Installation

_engine is a single CSS file. Drop it in and link it.

<link rel="stylesheet" href="_engine.css">

Or via npm:

npm install _engine.css
Important

The @property registrations must stay at the top level of the stylesheet. They cannot be nested inside @layer or @import. This is a CSS spec requirement.

Color engine

Every element in the DOM gets its background from a single oklch() expression applied via :where(*). Text color auto-derives from that background using a relative color function that flips at the perceptual midpoint.

The formula

--_bg: oklch( clamp(10%, calc( var(--_surf-base) + var(--surface) * var(--_surf-step) + var(--nudge-l) * var(--_l-gain) * 1% + var(--_interact) * var(--_dir) * 1% ), 99%) max(0, calc(var(--chroma) + var(--nudge-c) * var(--_c-gain))) calc(var(--hue) + var(--nudge-h) * var(--_h-gain)) );

Lightness is clamped between 10% and 99%. Chroma is floored at 0. Hue wraps naturally. The formula is additive — surface depth, nudge, and interaction state all stack.

Surface staircase

The .surface class auto-increments depth using a leapfrog parity system. Each nested .surface is one step deeper. Light mode steps darker, dark mode steps lighter.

.fill — knob preset

.fill is not a second formula. It's three property assignments that push the main formula into a vibrant range:

.fill { --surface: 0; --nudge-l: -30; --chroma: 0.14; }

Use on buttons and tags only. For vibrant cards, set --nudge-l and --chroma directly on the card.

.clickable — interaction states

.clickable sets --_interact on hover (-3) and active (-5). Because --_interact inherits, the entire subtree shifts together — background, text, tags, everything.

Nudge system

Three inheriting channel modifiers shift color relative to the inherited context. Set on any element and all descendants shift together. Reset with initial.

nudge-l: lightness

Negative = deeper, positive = lighter. --_l-gain flips direction in dark mode and scales 40% stronger (dark surfaces need bigger shifts to be perceptible).

Type system

--type is a single number. The formula computes pow(ratio, --type) and clamps between min/max viewport widths. Letter-spacing and line-height auto-adjust. The ratio widens from 1.2 (mobile) to 1.28 (desktop).

font-size: clamp( type-min * pow(min-ratio, --type), /* fluid interpolation */, type-max * pow(max-ratio, --type) ) * --cfg-type-scale;

Inline --type

The selector [style*="--type:"] activates the formula on any element. No class needed.

<span style="--type: 3;">Big number</span> <span style="--type: -2;">Tiny label</span>

Global scale

Set --cfg-type-scale on :root to uniformly scale all type. Default is 1.

Dark mode

--cfg-dark toggles between 0 and 1. Everything else is computed. The surface base inverts, the step direction reverses, and the directional multiplier --_dir flips from 1 to −1.

/* Automatic */ @media (prefers-color-scheme: dark) { :root { --cfg-dark: 1; } } /* Manual */ html[data-ui-scheme="dark"] { --cfg-dark: 1; } html[data-ui-scheme="light"] { --cfg-dark: 0; }

Per-channel gain

Tunable constants in core.config control how each channel responds to dark mode:

  • --_l-gain — light: 1.0, dark: −1.4. Flips direction, 40% stronger.
  • --_c-gain — light: 1.0, dark: 1.25. No flip, 25% boost (dark eats chroma).
  • --_h-gain — always 1. Hue is perceptually symmetric.
Tuning

If dark mode feels flat, increase the K constant on --_l-gain (currently 0.4). If over-corrected, pull it down. Same for --_c-gain (currently 0.25).

Page layouts

Three layout classes on <body>. All use CSS Grid. Children are targeted by id.

.page — scroll layout

A vertical flex column. #main gets flex: 1 to fill remaining space.

<body class="page"> <header id="header">...</header> <main id="main">...</main> <footer id="footer">...</footer> </body>

.dashboard — fixed viewport

A 3-column grid pinned to the viewport. #main scrolls independently.

<body class="dashboard"> <header id="header">...</header> <nav id="nav">...</nav> <main id="main">...</main> <aside id="aside">...</aside> <footer id="footer">...</footer> </body>

.sidebar-page — nav + content

Two-column grid with a sidebar nav and main content area.

Compositions

Reusable layout primitives. All accept --_gap for custom gap. Default gaps use spacing tokens.

.stack

Vertical flex column. Default gap: --space-s.

<div class="stack"> <div>Item 1</div> <div>Item 2</div> <div>Item 3</div> </div>

.row, .cluster

Horizontal flex with wrap. .row adds align-items: center. Default gap: --space-xs.

.split

Horizontal flex with justify-content: space-between. Push items to opposite ends.

.grid

CSS Grid with auto-fit. Column min-width controlled by --_col (default: 15rem).

<div class="grid" style="--_col: 20rem;"> <div class="card surface">...</div> <div class="card surface">...</div> <div class="card surface">...</div> </div>

.flank, .flank-end

Sidebar layouts. First child is the sidebar (--_flank sets basis, default 16rem). Last child fills remaining space. Wraps on narrow viewports. .flank-end reverses the order.

.center

Max-width container with inline padding. Override max with --_center-max.

Alignment utilities

.ai-start, .ai-center, .ai-end, .ai-baseline, .ai-stretch for align-items. .jc-start, .jc-center, .jc-end, .jc-between, .jc-around, .jc-evenly for justify-content. .as-* for align-self.

Buttons

The .btn class handles layout. Add .fill.clickable for primary, .surface.clickable for secondary, .ghost.clickable for tertiary.

Semantic variants via nudge

Color variants use --nudge-h — no separate classes. --nudge-h: 100 for success, --nudge-h: -20; --nudge-c: 0.04 for danger, --nudge-h: 60 for warning.

Utilities

Component helpers

  • .ripple — material-style ripple on hover/active. RTL-aware.
  • .truncate — single-line text overflow with ellipsis.
  • .clamp — multi-line clamp. Set --_lines (default 3).

Layout utilities

  • .mobile, .tablet, .desktop — responsive visibility.
  • .text-start, .text-center, .text-end — text alignment.
  • .full-bleed — break out of a centered container to full viewport width.
  • .vh — visually hidden but accessible to screen readers.
  • .np — hide when printing.

Layer order

The full cascade layer stack, in priority order:

@layer reset.fix, /* CSS spec bug fixes */ reset.opinion, /* Project-specific resets */ core.config, /* --cfg variables, media queries */ core.color, /* The one formula */ core.type, /* Fluid type scale */ core.spacing, /* Fluid spacing tokens */ theme, /* Font stacks, heading assignments */ layout.page, /* body.page, body.dashboard */ layout.composition, /* .stack, .row, .grid, .flank */ component.complex, /* .nav, .table, .tabs, .popover */ component.simple, /* .card, .btn, .pill, .input */ component.utls, /* .ripple, .truncate, .clamp */ utility.layout, /* .mobile, .text-center */ utility.exceptions, /* .vh */ utility.important; /* [hidden] !important */

Each layer is additive. Later layers can override earlier ones. Components in component.simple can override compositions in layout.composition. Utilities can override components.
