Get started

_engine is a property-driven CSS design system that derives an entire UI from a handful of registered custom properties. One formula computes every background. One formula computes every font size. Everything inherits, composes, and adapts to dark mode automatically.

It's built on @property registrations, OKLCH color math, and CSS pow() for fluid type — no JavaScript runtime, no build step, no preprocessor. The entire library is ~800 lines of vanilla CSS.

Installation

_engine is a single CSS file with zero dependencies. Drop it into your project and link it from your HTML. No package manager required, though you can install via npm if you prefer.

Recommended

Link the CSS file directly in your <head>. The @property registrations must be at the top level — they cannot be inside a @layer or @import.

<link rel="stylesheet" href="_engine.css">

Or install via npm:

npm install _engine.css

Core concepts

The system is built around five public knobs that control the entire visual output. Set them on any element and the engine computes everything downstream — background color, text contrast, font size, line height, letter spacing.

The five knobs

  • --hue — sets the color family (0–360). Inherits to all descendants.
  • --chroma — sets the saturation level. Low values are near-neutral, high values are vibrant.
  • --surface — staircase depth. Auto-incremented by .surface class via leapfrog parity.
  • --contrast — text contrast from 0 (invisible) to 1 (full). Use 0.7 for body, 0.45 for captions.
  • --type — type scale step. 0 = body, positive = larger headings, negative = smaller labels.

The nudge system

Three inheriting channel modifiers let you shift any element's color relative to the inherited context, without breaking out of the system:

  • --nudge-l — lightness. Negative = deeper, positive = lighter. Direction flips in dark mode via --_l-gain, and dark mode gets a 40% stronger response because dark surfaces need bigger shifts to be perceptible.
  • --nudge-c — chroma. Boost or reduce saturation locally. Dark mode compensates 25% via --_c-gain.
  • --nudge-h — hue. Rotate the hue relative to the inherited --hue. No direction flip — hue is perceptually symmetric.
Inheritance is the point

All three nudge properties use inherits: true. When you set --nudge-h: 60 on a card, every descendant — headings, body text, tags, buttons — shifts warm together. Use --nudge-*: initial to break out and reset to 0.

The formula

Every element in the DOM gets its background from a single oklch() expression on :where(*):

--_bg: oklch( clamp(10%, calc( var(--_surf-base) + var(--surface) * var(--_surf-step) + var(--nudge-l) * var(--_l-gain) * 1% + var(--_interact) * var(--_dir) * 1% ), 99%) max(0, calc(var(--chroma) + var(--nudge-c) * var(--_c-gain))) calc(var(--hue) + var(--nudge-h) * var(--_h-gain)) );

Text color auto-derives from --_bg using a relative color function that flips between light and dark text at the perceptual midpoint, scaled by --contrast.

Dark mode

Dark mode is a single number: --cfg-dark toggles between 0 and 1. Everything else is computed — the surface base flips, the surface step reverses, and the directional multiplier --_dir goes from 1 to −1 so that --nudge-l: -5 always means "5 units deeper" regardless of mode.

/* Automatic (follows OS) */ @media (prefers-color-scheme: dark) { :root { --cfg-dark: 1; } } /* Manual override via attribute */ html[data-ui-scheme="dark"] { --cfg-dark: 1; } html[data-ui-scheme="light"] { --cfg-dark: 0; }
Per-channel gain

The gain constants (--_l-gain, --_c-gain) are tunable in core.config. If dark mode feels flat, increase the K constant on lightness. If it feels over-corrected, pull it down. The current defaults are 0.4 for lightness and 0.25 for chroma.

Quick start

Here's a minimal page using the system:

<!DOCTYPE html> <html lang="en"> <head> <link rel="stylesheet" href="_engine.css"> </head> <body style="--hue: 250;"> <div class="card surface"> <h3>Hello _engine</h3> <p style="--contrast: 0.6;"> This card derives its background, text color, and font size entirely from the system. </p> <button class="btn fill clickable"> Action </button> </div> </body> </html>

Set --hue on the body. Add .surface to create depth. Use --contrast for text hierarchy. Use .fill.clickable on buttons. The engine handles everything else.
